## POST /api/orders

Create a new order from cart contents.

### Request

```http
POST /api/orders
Content-Type: application/json
```

```json
{
  "items": [
    {
      "productId": 42,
      "quantity": 2,
      "unitPrice": 29.99
    },
    {
      "productId": 7,
      "quantity": 1,
      "unitPrice": 149.99
    }
  ],
  "totalAmount": 209.97,
  "shippingAddress": {
    "name": "John Doe",
    "address": "123 Main Street, Apt 4B",
    "city": "Springfield",
    "state": "IL",
    "zipCode": "62701",
    "country": "USA"
  },
  "customerEmail": "john@example.com",
  "customerName": "John Doe"
}
```

**Field Descriptions**:

| Field | Type | Required | Constraints |
|-------|------|----------|-------------|
| `items` | `OrderItemRequest[]` | Yes | MinLength(1) |
| `items[].productId` | `int` | Yes | > 0, must exist in Products table |
| `items[].quantity` | `int` | Yes | >= 1 |
| `items[].unitPrice` | `decimal` | Yes | > 0.01 |
| `totalAmount` | `decimal` | Yes | > 0.01, must match sum of (qty × price) within $0.01 |
| `shippingAddress` | `AddressDto` | Yes | All fields required |
| `shippingAddress.name` | `string` | Yes | |
| `shippingAddress.address` | `string` | Yes | |
| `shippingAddress.city` | `string` | Yes | |
| `shippingAddress.state` | `string` | Yes | |
| `shippingAddress.zipCode` | `string` | Yes | |
| `shippingAddress.country` | `string` | Yes | |
| `customerEmail` | `string` | Yes | Valid email format |
| `customerName` | `string` | Yes | MinLength(2) |

### Response — 201 Created

```json
{
  "orderId": 1,
  "orderNumber": "ORD-2026-00001",
  "totalAmount": 209.97,
  "status": "Pending",
  "shippingAddress": {
    "name": "John Doe",
    "address": "123 Main Street, Apt 4B",
    "city": "Springfield",
    "state": "IL",
    "zipCode": "62701",
    "country": "USA"
  },
  "customerEmail": "john@example.com",
  "customerName": "John Doe",
  "items": [
    {
      "orderItemId": 1,
      "productId": 42,
      "productName": "Wireless Mouse",
      "quantity": 2,
      "unitPrice": 29.99,
      "subtotal": 59.98
    },
    {
      "orderItemId": 2,
      "productId": 7,
      "productName": "Mechanical Keyboard",
      "quantity": 1,
      "unitPrice": 149.99,
      "subtotal": 149.99
    }
  ],
  "createdAt": "2026-01-28T16:30:00Z",
  "estimatedDelivery": "2026-02-04T16:30:00Z"
}
```

**Response Fields**:

| Field | Type | Notes |
|-------|------|-------|
| `orderId` | `int` | Auto-generated primary key |
| `orderNumber` | `string` | Format: `ORD-YYYY-XXXXX` (generated by backend) |
| `totalAmount` | `decimal` | Echoed from request |
| `status` | `string` | Always `"Pending"` on creation |
| `shippingAddress` | `AddressDto` | Echoed from request (deserialized from JSON column) |
| `customerEmail` | `string` | Echoed from request |
| `customerName` | `string` | Echoed from request |
| `items[]` | `OrderItemDto[]` | Created items with server-generated IDs |
| `items[].orderItemId` | `int` | Auto-generated |
| `items[].productId` | `int` | From request |
| `items[].productName` | `string` | Looked up from Products table by backend |
| `items[].quantity` | `int` | From request |
| `items[].unitPrice` | `decimal` | From request |
| `items[].subtotal` | `decimal` | Computed: quantity × unitPrice |
| `createdAt` | `string` (ISO 8601) | Server timestamp |
| `estimatedDelivery` | `string` (ISO 8601) | Computed: createdAt + 7 days |

### Error Responses

#### 400 Bad Request — Model Validation

```json
{
  "type": "https://tools.ietf.org/html/rfc9110#section-15.5.1",
  "title": "One or more validation errors occurred.",
  "status": 400,
  "errors": {
    "Items": ["Order must contain at least one item"],
    "CustomerEmail": ["Invalid email address"]
  }
}
```

#### 400 Bad Request — Product Not Found

```json
{
  "error": "Products not found: 999, 888"
}
```

> Returned when one or more `productId` values do not exist in the Products table.

#### 400 Bad Request — Total Mismatch

```json
{
  "error": "Total amount mismatch. Expected: 209.97, Received: 200.00"
}
```

> Returned when `totalAmount` differs from the sum of (quantity × unitPrice) by more than $0.01.

#### 500 Internal Server Error

```json
{
  "error": "An error occurred while creating the order."
}
```

> Returned on database or transaction failure.

## GET /api/orders/{id}

Retrieve an order by its internal numeric ID.

### Request

```http
GET /api/orders/1
```

| Parameter | Type | Location | Notes |
|-----------|------|----------|-------|
| `id` | `int` | Path | Order's primary key |

### Response — 200 OK

Same schema as POST 201 response (see above).

### Error Responses

#### 404 Not Found

```json
{
  "error": "Order not found"
}
```

---

## GET /api/orders/number/{orderNumber}

Retrieve an order by its human-readable order number. **This is the endpoint used by the confirmation page on refresh/direct navigation.**

### Request

```http
GET /api/orders/number/ORD-2026-00001
```

| Parameter | Type | Location | Notes |
|-----------|------|----------|-------|
| `orderNumber` | `string` | Path | e.g., `ORD-2026-00001` |

### Response — 200 OK

Same schema as POST 201 response (see above).

### Error Responses

#### 404 Not Found

```json
{
  "error": "Order not found"
}
```

## Frontend Error Handling Contract

The `ApiError` class (`frontend/lib/api/client.ts`) exposes:
- `statusCode: number` — HTTP status code
- `message: string` — Error message from response body
- `response?: unknown` — Raw response (for model validation error objects)

The new `order-errors.ts` utility (to be created) maps these to user-friendly messages:

| Condition | User Message |
|-----------|-------------|
| 400 + message contains `"not found"` | "A product in your cart is no longer available. Please review your cart." |
| 400 + message contains `"mismatch"` | "Prices have changed since you added items to your cart. Please review and try again." |
| 400 (other) | "There was a problem with your order. Please review your information and try again." |
| 404 | "Order service is temporarily unavailable. Please try again." |
| 500 | "Something went wrong on our end. Please try again in a moment." |
| Network/timeout (no statusCode) | "Unable to reach the server. Please check your connection and try again." |
